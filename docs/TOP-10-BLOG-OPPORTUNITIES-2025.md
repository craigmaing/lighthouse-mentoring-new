## CONTENT FRAMEWORK (Apply to All 10 Posts)

### Structure (2,200-3,000 words each)

**1. Introduction (200 words)**
- Problem statement
- Why this matters now
- What reader will learn

**2. Main Definition Section (400 words)**
- What is [keyword]?
- Core responsibilities/deliverables
- How it differs from alternatives

**3. When You Need This (500 words)**
- Key scenarios
- Warning signs
- Decision framework

**4. What to Look For (600 words)**
- Essential credentials (mention Craig's: FCMI, FRSPH, IoD)
- Experience requirements
- Red flags to avoid

**5. Process & Approach (400 words)**
- How it works
- Typical timeline
- Expected outcomes

**6. Investment & ROI (300 words)**
- Pricing ranges (be transparent)
- ROI metrics
- Value justification

**7. FAQ Section (400 words)**
- 5-7 common questions
- Use FAQ schema markup

**8. Conclusion + CTA (200 words)**
- Summary
- Next steps
- Strong CTA

### SEO Optimization

**Title Tag Formula:**
`[Keyword]: [Benefit] ([Year] Complete Guide)`

**Example:**
"Business Transformation Consultant: When and How to Hire (2025 Complete Guide)"

**Meta Description Formula (155 chars):**
`[Keyword] explained by [Credential]. Learn [benefit]. [Authority signal]. [CTA].`

**Example:**
"Business transformation consultant guide by FCMI Fellow. Learn when to hire, what they do, and expected ROI. 17 years board experience. Read free guide."

### Internal Linking Strategy

**Every post should link to:**
- 2-3 service pages (relevant to keyword)
- 1-2 other blog posts (related topics)
- About page (Craig's credentials)
- Contact/booking page

**Example for Business Transformation post:**
- Link to: Management Consulting service page (when created)
- Link to: Strategic Consultant blog post
- Link to: Change Management blog post
- Link to: About page (FCMI Fellow mention)
